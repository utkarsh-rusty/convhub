# README screenshots

Drop PNG screenshots in this folder using these exact filenames so they appear on GitHub:

| File | Suggested content | Suggested caption |
|------|-------------------|-------------------|
| `landing.png` | Marketing / landing hero | ConvHub landing — continue a teammate’s AI session |
| `conversation.png` | Shared conversation with streaming | Shared conversation with realtime collaboration |
| `branching.png` | Branch manager or branch tree | Conversation branching and lineage |
| `commit-graph.png` | Commit graph | Commit graph across a conversation family |
| `dashboard.png` | App dashboard / home | Projects and recent conversations |
| `providers.png` | AI providers settings | Owned AI providers and routing |
| `repository.png` | Repository page (memory / pull package cards) | Coding repository — memory, pull package, handoff |
| `handoff.png` | Claude Handoff preview or downloaded Markdown | Claude Handoff document ready to paste |

Tips:

- Prefer wide captures (about 1200–1600px wide).
- Use PNG or JPEG; update `README.md` paths if you change the extension.
- Paths in the root `README.md` are relative: `docs/images/<name>.png`.
- Until assets exist, README still references these paths as placeholders.
